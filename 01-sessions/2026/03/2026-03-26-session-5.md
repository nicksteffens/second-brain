### What We Accomplished
- Investigated deploy failures on PR #10504 (Node 24 prototype) in the front-end repo
- Traced failure to root cause: `util.isRegExp is not a function` in `clean-css@3.4.28` — API removed in Node 22+
- Identified why build passes but deploy fails: CSS minification only enabled in production-like builds (`ember deploy`), not CI build step
- Documented full root cause analysis as a PR comment on movableink/front-end#10504

### Challenges Encountered
- Cross-repo context: failure in front-end repo while working from front-end-infra

### Key Insight
Production-only code paths (CSS minification during `ember deploy`) can hide Node compat issues from CI build/test steps. Always verify Node upgrades against production build paths.
