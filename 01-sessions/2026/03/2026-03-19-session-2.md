### What We Accomplished
- Researched GitHub Actions Node 20 deprecation timeline (June 2, 2026 default Node 24, fall 2026 removal)
- Traced the full `esm` → `broccoli@3.5.2` → `ember-cli 3.28` blocking chain causing SIGABRT on Node 22+
- Audited all Ember apps across org: 7 packages (4 apps + 3 addons), all on Ember 3.28.x
- Confirmed railsapp (Rails), ui (React/MUI), and other repos are not Ember
- Created yarn patch for `broccoli@3.5.2` removing `esm` shim — mirrors broccoli PR #531
- Swapped `@mozaic-ds/postcss-sass` → `@csstools/postcss-sass@^5.1.1` (source-map WASM fix)
- Bumped `ember-auto-import` to `^2.12.1`, added `webpack@^5`
- Upgraded Yarn 3.1.1 → 4.13.0 (3.1.1 crashes on Node 24 CI runners)
- Validated build/lint/tests on Node 22 AND Node 24 — all green (85 tests, 84 pass, 1 pre-existing skip)
- CI passes on PR #187 (movableink/fluid) — lint, test, Percy all green
- Created Notion doc: "Node 22/24 Upgrade: Ember App Feasibility & Patching Strategy"
- Updated Shortcut sc-190048 with full inventory and GitHub Actions deprecation context
- Discovered Storybook 6 / webpack 4 needs `--openssl-legacy-provider` on Node 24 (separate upgrade)

### Challenges Encountered
- Yarn 3.1.1 crashes on Node 24 CI runners with `isDate is not a function` — Yarn's internal date handling uses removed Node APIs. Fixed by upgrading to Yarn 4.13.0

### Most Valuable Collaboration
Deep dependency chain research — tracing the esm → broccoli → ember-cli chain, reading the actual source code of `load_brocfile.js` and `@ember-data/private-build-infra`, and identifying that a 4-line yarn patch eliminates the Node 22+ blocker without any Ember upgrade.

### Key Insight
A targeted yarn patch removing the dead `esm` shim is far simpler than upgrading Ember — the same patch applies to all 7 Ember packages across the org. Node.js 20.19+ has native `require(esm)` support, making the shim unnecessary. The `esm` package is a corpse in the dependency tree.
